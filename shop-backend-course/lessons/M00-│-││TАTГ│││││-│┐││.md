# M00 — Окружение и структура проекта

> **Цель:** поднять чистое окружение, запустить первый FastAPI-сервер и
> заложить структуру, в которой будем расти весь курс.

---

## 🟦 Теория — что такое backend

Когда ты открываешь сайт, твой браузер (**клиент**) шлёт **HTTP-запрос** на
сервер. Сервер — это программа, которая принимает запрос, что-то делает (лезет
в базу, считает, проверяет права) и возвращает **HTTP-ответ**. Наш бэкенд
магазина — как раз такая программа.

Каждый HTTP-запрос состоит из:

- **Метод** — что делаем: `GET` (получить), `POST` (создать), `PUT/PATCH`
  (изменить), `DELETE` (удалить).
- **Путь** — например `/products/42`.
- **Заголовки** — метаданные (кто я, какой формат принимаю).
- **Тело** — данные (например JSON нового товара).

В ответ сервер шлёт **статус-код** (`200` ок, `404` не найдено, `401` не
авторизован, `500` ошибка сервера) и тело ответа.

> **Почему FastAPI.** Это современный Python-фреймворк: он асинхронный (держит
> много одновременных запросов), проверяет типы данных за тебя и сам генерирует
> интерактивную документацию. FastAPI работает по стандарту **ASGI**
> (асинхронный), а запускает его сервер `uvicorn`.

## 🟧 Методология — как профи начинает проект

Четыре правила, которые экономят месяцы боли:

1. **Изолируй зависимости.** Каждый проект — в своём виртуальном окружении
   (`.venv`), чтобы библиотеки не конфликтовали между проектами.
2. **Фиксируй версии.** Записывай, какие версии библиотек стоят, чтобы проект
   собрался у любого и через год.
3. **Сразу git.** Версионирование с первого коммита, а не «потом добавлю».
4. **Понятная структура с первого дня.** Не сваливай всё в один файл —
   раскладывай по слоям (роуты / модели / схемы / логика).

## 🟩 Практика — поднимаем проект

**Шаг 1.** Проверь Python (нужен 3.11+, берём 3.13):

```bash
python --version    # Python 3.13.x
```

**Шаг 2.** Создай папку и виртуальное окружение:

```bash
mkdir shop-backend && cd shop-backend
python -m venv .venv
# активировать: Linux/macOS
source .venv/bin/activate
# Windows (PowerShell)
.venv\Scripts\Activate.ps1
```

После активации в начале строки терминала появится `(.venv)` — значит окружение
включено.

**Шаг 3.** Установи FastAPI (вариант `[standard]` сразу тянет uvicorn и нужные
мелочи):

```bash
pip install "fastapi[standard]"
```

**Шаг 4.** Заложи структуру проекта — мы будем наполнять её весь курс:

```
shop-backend/
├── app/
│   ├── __init__.py
│   ├── main.py          # точка входа: создаём приложение
│   ├── api/             # роуты (эндпоинты)
│   ├── core/            # конфиг, настройки, безопасность
│   ├── models/          # таблицы БД (SQLAlchemy)
│   ├── schemas/         # Pydantic-схемы (валидация)
│   └── services/        # бизнес-логика
├── tests/
├── .gitignore
└── requirements.txt
```

**Шаг 5.** Самый первый сервер — `app/main.py`:

```python
from fastapi import FastAPI

app = FastAPI(title="Shop API", version="0.1.0")


@app.get("/health")
async def health():
    return {"status": "ok"}
```

**Шаг 6.** Запусти сервер:

```bash
uvicorn app.main:app --reload
```

Флаг `--reload` перезапускает сервер при изменении кода. Открой в браузере:

- `http://127.0.0.1:8000/health` — увидишь `{"status":"ok"}`;
- `http://127.0.0.1:8000/docs` — **автоматическая** интерактивная документация
  (Swagger UI). Её FastAPI собрал сам.

**Шаг 7.** Зафиксируй версии и подключи git — `.gitignore`:

```
.venv/
__pycache__/
*.pyc
.env
```

```bash
pip freeze > requirements.txt
git init && git add . && git commit -m "init: FastAPI skeleton"
```
