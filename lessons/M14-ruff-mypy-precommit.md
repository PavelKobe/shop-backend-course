# M14 — Качество кода: Ruff, Mypy, pre-commit

> **Цель:** автоматически держать код чистым, единообразным и типобезопасным —
> чтобы машина ловила ошибки до того, как они попадут в репозиторий.

---

## 🟦 Теория

Три инструмента наводят порядок без ручного труда:

- **Ruff** — сверхбыстрый линтер и форматтер. **Линтер** ищет проблемы
  (неиспользуемый импорт, подозрительная конструкция), **форматтер** приводит код
  к единому стилю (отступы, кавычки, длина строк). Заменяет связку
  flake8 + isort + black в одном инструменте.
- **Mypy** — проверка типов. Python динамический, но мы пишем аннотации
  (`name: str`). Mypy проверяет, что ты не передал число туда, где ждут строку —
  и находит баги до запуска.
- **pre-commit** — менеджер git-хуков. Перед каждым коммитом он автоматически
  прогоняет Ruff и Mypy. Если есть проблемы — коммит не пройдёт, пока не починишь.

> **Зачем это новичку?** Эти инструменты — твой бесплатный код-ревьюер. Они учат
> хорошему стилю и ловят целый класс ошибок ещё до того, как ты их заметишь.

---

## 🟧 Методология

- **Стиль задаётся конфигом, а не спорами** — все правила в `pyproject.toml`.
- **Одинаковые проверки локально и в CI** — pre-commit и сервер запускают одно и
  то же, расхождений нет.
- **Чини замечания сразу** — не копи технический долг.
- **Автоформат при сохранении** — настрой VS Code, чтобы Ruff форматировал файл
  на `Ctrl+S`.

## 🟩 Практика

**Шаг 1.** Установи:

```bash
pip install ruff mypy pre-commit
```

**Шаг 2.** Настрой Ruff и Mypy — `pyproject.toml` в корне:

```toml
[tool.ruff]
line-length = 88
target-version = "py313"

[tool.ruff.lint]
# E,F — ошибки/пифлейк; I — сортировка импортов; UP — апгрейд синтаксиса;
# B — баги; SIM — упрощения
select = ["E", "F", "I", "UP", "B", "SIM"]

[tool.mypy]
python_version = "3.13"
plugins = ["pydantic.mypy"]
ignore_missing_imports = true
check_untyped_defs = true

[tool.pytest.ini_options]
asyncio_mode = "auto"
testpaths = ["tests"]
```

**Шаг 3.** Прогони вручную, чтобы увидеть, что они делают:

```bash
ruff check .            # найти проблемы
ruff check . --fix      # часть починить автоматически
ruff format .           # отформатировать
mypy app                # проверить типы
```

Скорее всего, Ruff что-то поправит (порядок импортов, лишние пробелы), а Mypy
подскажет, где не хватает аннотаций. Почини замечания — это и есть обучение.

**Шаг 4.** Автоматизируй через pre-commit — `.pre-commit-config.yaml`:

```yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.8.0
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format
  - repo: https://github.com/pre-commit/mirrors-mypy
    rev: v1.13.0
    hooks:
      - id: mypy
        additional_dependencies: [pydantic]
```

Установи хуки в git:

```bash
pre-commit install
```

Теперь при каждом `git commit` проверки запускаются автоматически. Прогнать по
всем файлам сразу:

```bash
pre-commit run --all-files
```

**Шаг 5.** Настрой VS Code (`.vscode/settings.json` — он в `.gitignore`, поэтому
сделай локально на каждом ПК):

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "charliermarsh.ruff",
  "[python]": {
    "editor.codeActionsOnSave": { "source.fixAll.ruff": "explicit" }
  }
}
```

Теперь код форматируется при сохранении, а импорты сортируются сами.

---

> **Итог модуля.** Качество кода держится автоматически: Ruff форматирует и ловит
> проблемы, Mypy проверяет типы, pre-commit не пускает «грязные» коммиты. Остался
> финал — собрать всё в продакшен-конфигурацию и запустить одной командой.
